kts.SetRespawnType

Name

kts.SetRespawnType – set the current "respawn type" for all knights

Synopsis

kts.SetRespawnType(respawn_type)

Description

This function controls respawn behaviour. The respawn_type parameter must be one of the following strings:

  • "normal" – the normal Knights respawn behaviour. Knights will spawn at a fixed entry point (or "home"), which only changes if the wand of securing is used.
  • "different" – Knights will appear at a different, randomly selected "home" tile (dungeon entry point) each time they respawn. Knights can still heal by visiting their most recent entry point.
  • "anywhere" – The "home" (entry point) tiles are not used for respawning at all. Instead knights just respawn at a completely random dungeon position. Healing by returning to one's entry point will not be available.

Return Value

No value is returned.

Errors

If the input parameter is not one of the strings listed above, an error will result.

Notes

If both SetRespawnType and SetRespawnFunction are used, then the "respawn function" takes priority, and the "respawn type" is only used if the respawn function returned nil.
